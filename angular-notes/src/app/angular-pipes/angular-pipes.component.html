Angular Pipes

Pipes are simple functions to use in templates to accept an input value and return a transformed value.

Built-in Pipes

  1. Upper case pipe

    {{ ucasepipe | uppercase }}

  2. Lower case pipe

    {{ lcasepipe | lowercase }}

  3. Currency pipe

    {{ currencypipe | currency : "INR" }}

    {{ 1234.567 | currency:'GBP':'code' }}

    {{ 1234.567 | currency:'USD':'symbol':'1.0-0' }}

    {{ 1234.567 | currency:'USD':'symbol':'1.1-2' }}

    {{ 1234.567 | currency:'USD':'symbol':'1.2-2' }}

  4. Percentage Pipe Formatting Types

    {{ percentagepipe | percent }}

    {{ 0.1234 | percent : "1.0-0" }}

    {{ 0.1234 | percent : "1.1-1" }}

    {{ 0.1234 | percent : "1.1-2" }}

  5. Json Pipe

    Json Pipe:{{ jsonpipe | json }}

  6. Slice pipe

    {{ slicepipe | slice : 1 : 4 }}

  7. Date Pipe Formatting Types

    Default format: {{ datepipe | date }}

    shortDate: {{ datepipe | date : "shortDate" }}

    mediumDate: {{ datepipe | date : "mediumDate" }}

    longDate: {{ datepipe | date : "longDate" }}

    fullDate: {{ datepipe | date : "fullDate" }}

    Custom format: {{ datepipe | date : "yyyy-MM-dd HH:mm:ss" }}

  8. Number Pipe Formatting types

    {{ 1234.5678 | number }}

    {{ 1234.5678000000 | number : "8.6-8" }}

    {{ 1234.5 | number : "1.2-2" }}

    {{ 1234 | number : "5.1-5" }}

Custom Pipe(Reverse a String)

Original: "{{ 'hello world' }}"

Reversed: "{{ 'hello world' | reverse }}"

Services in Angular

Services allow your code to share common functionality across the application. Create a single reusable service and inject it into the components that need it, which makes your code clean. An Angular service is simply a JavaScript function, including its related properties and methods which can perform a particular task or a group of tasks. Actually, service is a mechanism to use shared responsibilities within one or multiple components. Services are a piece of code that are used to perform a specific task, a service can contain a value or function or combinations of both. Services are injected into application using dependency injection mechanism. Services prevent us from writing the same code at multiple sections of our application. The best solution is to write services are injecting in application where we need it.
